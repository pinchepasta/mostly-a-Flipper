**2026-07-02: BLE Spam works but only ONCE per boot — root cause = BLE controller deinit leak.** After the UUID-DB-skip fix, spam's first attack works, but the 2nd refuses/crashes. Instrumented `ble_spam_hal.c` with heap logs; captured a real cycle:
  - app entry (DB skipped): 100740 free → 1st `ble_spam_hal_start` OK → `HAL ready free=37772` (spam runs) → `HAL stopped free=60860 largest=31744`.
  - 2nd start: needs `BLE_SPAM_MIN_FREE_INTERNAL=73728`, only 60860 → refuse → the 4 radio scenes' `previous_scene()`-from-`on_enter` retry-loop → GUI lockup/"crash".
  - **Each `esp_bt_controller`+`esp_bluedroid` init/deinit cycle leaks ~13-20 KB** (never returns to the ~74 KB it started from). This is an ESP-IDF/no-PSRAM Bluedroid-deinit limitation, not fixable in our code; also the heap is fragmented (`largest=31744` while total=60860).
- **Fix (keep the radio up across attacks instead of cycling it):** `ble_spam_hal.c` — `s_hal_started` guard makes `ble_spam_hal_start()` idempotent (reuse if up) and `ble_spam_hal_stop()` a no-op if already down. `scene_running.c` on_exit now calls `ble_spam_hal_stop_adv()` (stop advertising, KEEP controller) instead of `ble_spam_hal_stop()`. `scene_main.c` on_enter calls `ble_spam_hal_stop()` (full teardown when leaving the spam section back to main). `ble_spam_app.c` calls `ble_spam_hal_stop()` after `view_dispatcher_run` as an app-exit safety net. Net: controller inits once per spam *session* (Main→Spam→…→Main), so repeated attacks within a session reuse it and don't leak. Built + flashed. Heap-log lines in start/ready/stop left in for now.
**2026-07-02 (cont.): leak is ONE-TIME, not per-session — fixed by lowering the RAM gate; then fixed Walk + BT-on false-negative.** Serial capture across many spam sessions proved: within a session free is FLAT (~37,760 at HAL-ready every attack) and every teardown returns to the SAME ~60,808 free. **2026-07-04: BOTH the sound and deauth bugs root-caused and fixed (built clean; deauth verified at binary level, sound pending an audible test — user couldn't hear at fix time).**
- **Sound (ES8311) real cause = wrong MCLK source, NOT the reg sequence being absent.** The 2026-07-02 init used `reg 0x01=0x30` (codec expects MCLK on its MCLK pin) + I2S drove MCLK on GPIO0. But the ADV routes NO MCLK to the codec. Authoritative proof from `m5stack/M5Unified` `src/M5Unified.cpp`: `_speaker_enabled_cb_cardputer_adv` uses `0x01=0xB5` (**MCLK from BCLK/SCLK**), and the `board_M5CardputerADV` spk_cfg sets only pin_bck=41/pin_ws=43/pin_data_out=42 — it deliberately omits `pin_mck`. So codec had no clock → DAC never ran → silence even though it ACKs @0x18. The **NS4150B PA is always-on (no SW enable GPIO)** per M5Unified, so PA-enable was a red herring. **Fix** ([furi_hal_speaker.c](components/furi_hal/furi_hal_speaker.c)): replaced the 27-reg hand-rolled `es8311_seq` with M5Unified's known-good 9-reg playback set (`00=1F` reset, `00=80`, `01=B5`, `02=18`, `0D=01`, `12=00`, `13=10`, `32=BF`, `37=08`); set I2S `.mclk = I2S_GPIO_UNUSED` (was BOARD_PIN_I2S_MCLK=GPIO0, which is the BOOT strap — useless + wrong). `BOARD_PIN_I2S_MCLK` define now unused (left as doc).

PATHS "$ENV{HOME}/.espressif/tools/xtensa-esp-elf/*/.../bin" NO_DEFAULT_PATH)` had THREE defects: (1) `$HOME/.espressif` doesn't exist here — tools are at `C:\Espressif` (`IDF_TOOLS_PATH`); (2) `find_program` does NOT expand a literal `*` in PATHS; (3) `NO_DEFAULT_PATH` blocked the PATH fallback (toolchain IS on PATH during a build). So `_xt_ar`/`_xt_objcopy` were NOTFOUND, the patch was **silently skipped (no warning)**, `ieee80211_raw_frame_sanity_check` stayed intact → `esp_wifi_80211_tx` rejected every deauth(0xC0)/disassoc(0xA0) → `s_frames_sent` stuck at 0. **Verified** by disassembling the installed `libnet80211.a` (unpatched: `entry a1,64; extui...` reject path). **Fix** ([CMakeLists.txt](CMakeLists.txt)): `file(GLOB _xt_bindirs ...)` to expand the versioned dir, `find_program(... HINTS ${_xt_bindirs})` WITHOUT NO_DEFAULT_PATH, probe `$ENV{IDF_TOOLS_PATH}` + `$ENV{HOME}/.espressif`, invoke the patch via `python`, and emit a loud `message(WARNING ...)` when skipped.
